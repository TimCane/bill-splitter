# CI

GitHub Actions. Philosophy: every PR proves the contracts still hold in a
few minutes; the expensive full-stack e2e runs only where it pays.

## Workflows

### `ci.yml` - on every PR to `main` and on `main`

Jobs (parallel where independent):

| Job | Steps | Blocks merge |
| --- | --- | --- |
| `backend` | `dotnet restore` -> `build -warnaserror` -> unit tests -> integration tests (Testcontainers; Docker is available on `ubuntu-latest`) | yes |
| `frontend` | `pnpm install --frozen-lockfile` -> `prettier --check` -> `tsc --noEmit` -> `vitest run` -> `vite build` | yes |
| `ocr` | `ruff check` -> `pytest` (unit tests of the FastAPI layer with PaddleOCR mocked - the real model never loads in CI) | yes |
| `contract` | build backend -> generate OpenAPI json -> NSwag regenerate `client.ts` -> `git diff --exit-code`; also diff the shared DTO fixtures | yes |
| `e2e` | compose up full stack (prebuilt OCR image from GHCR, below) -> Puppeteer specs | only when labeled `e2e` or on `main` |

Path filters: each job also runs when its own tree or `docs/` contracts
change; `contract` runs whenever backend or frontend change.

### `ocr-image.yml` - on changes under `ocr/`

Builds the OCR image (slow: PaddlePaddle wheels) and pushes to GHCR
`ghcr.io/timcane/bill-splitter-ocr:{sha,latest}`. The e2e job and
production deploys consume this image instead of rebuilding it - nothing
else in CI should ever pay the 2GB build.

## Caching

- NuGet: `actions/cache` on `~/.nuget/packages` keyed by
  `Directory.Packages.props`.
- pnpm: `actions/setup-node` + pnpm store cache keyed by
  `pnpm-lock.yaml`.
- Docker: GHA buildx layer cache for the OCR image.

## Merge policy

- Branch protection on `main`: required checks `backend`, `frontend`,
  `ocr`, `contract`; linear history (squash merges only,
  matching house PR style).
- No auto-deploy from CI in MVP - deploys are manual
  ([13-deployment.md](13-deployment.md)). Revisit once the app is hosted
  somewhere real.
